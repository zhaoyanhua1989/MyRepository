1、ListView： item单个显示可以用android.R.layout.simple_list_item_1，Adapter用ArrayAdapter item单个显示并且可以勾选，用android.R.layout.simple_list_item_multiple_choice item两个显示可以用android.R.layout.simple_list_item_2，Adapter用SimpleAdapter item用自己定义的，包括不止textView显示时，需要自己定义Adapter继承BaseAdapter来显示(涉及sqlite查询时可用SimpleCursorAdapter) 2、分块显示： 自定义让adapter实现Sectionindexer接口，重写getPositionForSection(int sec)和 getSectionForPosition(int pos)接口。 3、分块与导航双功能显示： 两个listView，根据position和section位置关联显示，listView.setSelectionFromTop(int position, int y) 4、ListActivity可以实现ListView效果. 5、ExpandableListView： 类似于QQ的显示效果，外层是一个ListView，点击其中一个item，显示子ListView 简单实现案例：两个数组，一个String groups[],一个String childs[][]，自定义adapter继承 BaseExpandableListAdapter，重写起对应方法。 6、AutoCompleteTextView： 自动填充textView。默认输入两个字符就下拉提示输入内容，类似百度搜索提示。 简单实现案例：一个数组，用ArrayAdapter显示。 7、Spinner： 下拉选择框。 简单实现案例：一个数组，用ArrayAdapter显示。 8、GridView： 简单实现案例：其中加载图片，ImageView直接反射创建，LayoutParams从GridView的父类AbsListView获取。 其中图片需要压缩：Options对象来获取压缩比例，然后decodeResource(getResources(), int id, Options opts) 9、自定义view的UI： 以button为例 1、background设置为：@drawable/btn_press_select； 2、创建drawable目录创建selector类型xml，定义文件名btn_press_select； 3、btn_press_select添加item设定state和对应的drawable=@drawable/btn_press_true； 4、在drawable目录下创建shape类型xml命名btn_press_true，添加背景颜色(solid android:color)、 圆角(corners android:radius="2dp")、描边(stroke android:width、color)等。 10、CheckBox(复选框)： 文字和勾选框，isChecked()判断有没有被勾选，默认勾选框在左边，无法改变勾选框位置。 12、RadioGroup(RadioButton单选框)： 一般是添加RadioButton做选择，单选，通过getCheckedRadioButtonId()获取选择的button的id。 RadioButton通过设置button值为@null删除原有背景图片，设置drawable***值设置图片，设置padding值改变内边距 13、ScrollView(垂直滚动)、HorizontalScrollView(水平滚动). 14、Dialog(对话框)： 1、AlertDialog(提示对话框)： 创建：new AlertDialog.Builder(context).setIcon(图片).setTitle(标题).setMultiChoiceItems(选择描述String[]， 默认选择boolean[]，监听器).setPositiveButton(确定，监听器).setNegativeButton(取消，监听器).create().show(); 2、ProgressDialog(进度对话框)：progressDialog.setProgress(int); 3、DatePickerDialog(日期对话框)： DatePickerDialog dialog = new DatePickerDialog(context, 监听, 年, 月, 日); 4、TimePickerDialog(时间对话框)： TimePickerDialog dialog = new TimePickerDialog(context, 监听, 时, 分, boolean is24HourView); 注意：在安卓4.4到5.0之间系统的手机上，AlertDialog的顶部可能被剪切掉，需要设置dialog属性： dialog.getWindow().setFlags(WindowManager.LayoutParams.FLAG_FULLSCREEN, WindowManager.LayoutParams.FLAG_FULLSCREEN); 15、Activity中构建菜单： 1、重写onCreateContextMenu()：上下文菜单，需要长按，创建菜单示例代码如下： menu.add(0, 1, 100, "查看"); //参数：组ID(一个组？),itemId(定位是哪个按钮的关键id),顺序,标题 menu.add(0, 2, 102, "编辑"); menu.add(0, 3, 103, "删除"); 监听： onContextItemSelected()：item.getItemId()==1/2/3 2、重写onCreateOptionsMenu()：创建右上角菜单，创建菜单示例代码如下： 1)、系统菜单： getMenuInflater().inflate(R.menu.main, menu); //在res/menu/下面有系统菜单main.xml 2)、自定义菜单： 反射构建menu菜单显示内容(在onCreateOptionsMenu生命周期中，例)： MenuItem item = menu.add(Menu.NONE, 100, 102, "自拍"); item.setIcon(android.R.drawable.ic_menu_camera); item.setShowAsAction(MenuItem.SHOW_AS_ACTION_ALWAYS); 监听： onOptionsItemSelected()：item.getItemId()==100 3、创建PopupMenu：弹出式菜单，不需要长按，示例代码如下： new PopupMenu(context, 依赖的view).getMenuInflater().inflate(R.menu.pop, popup.getMenu()); 需要在res/menu/下面创建菜单pop.xml 监听： popup.setOnMenuItemClickListener(new PopupMenu.OnMenuItemClickListener() {...}); 4、添加子菜单： Menu.addSubMenu() 16、自定义view，需继承安卓的view(View/EditText/RelativeLayout等)： 1、可以什么都不重写，只实现自己的逻辑：比如动画效果； 2、在自定义view类代码中new Paint并重写onDraw()方法实现自己的view的UI。 自定义view，不继承安卓的view，自定义background，设置selector和shape实现。 17、Activity设置窗口模式： 1、用安卓的：android:theme="@android:style/Theme.Dialog" 2、自定义：@style/Theme.FloatActivity (需要在style中自定义style name="Theme.FloatActivity") 18、设置style时： 19、Activity: finish后，Activity不会马上被销毁，打印this不为空，对象还存在。
